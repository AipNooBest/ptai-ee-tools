Режим анализа уязвимостей PT AI. Определяет будет ли сборочный конвейер дожидаться завершения анализа или ограничится запуском задачи сканирования и вернет управление в конвейер

Возможные варианты:

  • Асинхронный режим
    Конвейер сборки запустит анализ безопасности приложения (AST) и вернется к исполнению следующих шагов. Единственный файл результатов, который публикуется в папке рабочей области CI, — это result.url, который содержит конечную точку REST API PT AI для получения результатов сканирования
  • Синхронный режим
    Конвейер сборки запускает анализ безопасности приложения и ожидает его завершения. На последнем этапе задания AST PT AI генерирует отчеты и проводит оценку политики безопасности (если таковая имеется). Этот режим работы позволяет вам определить, следует ли помечать шаг построения CI как неудачный, если оценка политики не удалась, а также определить отчеты, которые будут создаваться после завершения сканирования
